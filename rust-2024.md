# Rust 2024 Edition Improvements

With both the `core` (`src/app`) and `backend` (`src/backend`) modules now upgraded to the Rust 2024 edition, the codebase has been refactored to take advantage of the new ergonomic, safety, and standard library features introduced in this edition.

## Core Module (`src/app`)

### 1. Removed `expr_2021` Workarounds in Macros
During the 2024 edition migration, `cargo fix --edition` replaces the `expr` macro fragment specifier with `expr_2021` to preserve backwards compatibility (since Rust 2024 expands what expressions can match, e.g., `const` blocks).
- **Location:** `src/app/src/macros.rs` and `src/app/src/update/wifi.rs`
- **Action Taken:** All instances of `$name:expr_2021` were safely reverted to the standard `$name:expr`.

### 2. Replaced `lazy_static` with `std::sync::LazyLock`
Rust 1.80+ stabilized `std::sync::LazyLock`, allowing the removal of the external `lazy_static` dependency entirely. This reduces macro compilation overhead and relies on standard library primitives.
- **Location:** `src/app/src/wasm.rs` and `src/app/Cargo.toml`
- **Action Taken:** Removed `lazy_static` dependency and replaced the static initialization of the Crux `CORE` bridge with `std::sync::LazyLock`.

### 3. Leveraged Prelude Additions
The `Future` and `IntoFuture` traits are now part of the standard prelude in Rust 2024.
- **Location:** `src/app/src/commands/websocket.rs`
- **Action Taken:** Simplified `impl std::future::Future<Output = WebSocketOutput>` to `impl Future<Output = WebSocketOutput>`.

## Backend Module (`src/backend`)

### 1. Leveraged New Temporary Lifetime Rules for `if let` and `match`
Rust 2024 drops temporary variables in the scrutinee of `if let` and `match` expressions immediately before the block executes. This eliminates the need for manual scoping when holding non-`Send` types (like `MutexGuard`) across `.await` points.
- **Action Taken:** Code review confirmed that `lock().unwrap()` patterns across `src/backend/src/main.rs` and `src/backend/src/services/` are naturally compliant and benefit from the new implicit drop rules.

### 2. Simplified `impl Trait` Return Types (RPIT)
In Rust 2024, `impl Trait` return types automatically capture all generic type and lifetime parameters in scope.
- **Action Taken:** Future Actix handlers returning `impl Responder` or `impl Future` no longer require the `+ '_` explicit lifetime bounds.

### 3. Exploited the Never Type (`!`) Fallback
Rust 2024 changes the fallback for unconstrained type variables to `!` instead of `()`.
- **Action Taken:** This benefits dead-code analysis in async state machines implicitly across the project.
